UI/UX Design

Here are some examples of some prototypes of apps I've created during my time at university. These have all been designed on InVision, and have both had a full design process - from wireframes and mood boards to a functional prototype.

Moods.

Moods. is a progressive web application that was designed from wireframes, to prototype and finally to a functioning website.

It's purpose is to be a mental health companion app, which people can use to find support for anxiety, stress, sadness, anger, and happiness. It provides information and advice for those who may be suffering from the listed problems, and also features built-in applications to help them - such as a to-do app, meditation app, anxiety, and more.


Student Links

Student Links is a complete prototype that I designed on InVision, which was created for one of my assignments to create a unique application that would be useful.

The app is designed to offer support and guidance to university students in their city. It provides support for student living and health, locations of things to do and places students may need, social media competitions that encourage healthy living and budgeting, and a student forum, which allows students to offer suggestions for the app and find further support.


Websites

These show some of the websites that I have created for both university and for different organisations as freelance projects.

All of these sites (including this portfolio site) have been coded using HTML, CSS and JavaScript to be fully functional on any device, with the aim of achieving a good design and an easy UI.

Moods. Site

This site was created after the orignal design of its prototype.

It is designed as a progressive web application, so features a slightly different UI design to my other sites, as well as featuring multiple, built-in applications, such as a to-do list which uses local storage, a weather app which uses API and more.

Awareness Sites

These two sites were created as part of my Technology Project (otherwide known as a dissertation) at DMU.

Overall these sites are relatively basic, using a less JavaScript, but plenty of HTML and CSS to ensure they're fully adaptive to screen size. These sites were combined with social media campaign to help raise awareness of Children's Mental Health and University Student Mental Health.

This project was successful in raising awareness and allowed me to focus on ensuring the site was fully functional over creating something with lots of built-in features, although I did have to ensure each site had something extra to make it stand out.

Adhar Project

This site was my first ever freelance project, where I was tasked with re-creating Adhar Project's website.

Adhar's previous website was extremely outdated and created using an old web-design platform, so I completely coded the site from scratch and followed their desired design.

The charity didn't want any fancy features or a particularly bold design, they just wanted an simple, informational site that was good for their audience.

Regardless, this project helped me learn new skills and ways to learn independently, and I have since been able to develop my knowledge of codiing far beyond this.

Global Heads

This was a small, single page website that was produced for Global Heads - a startup company that has the plans of relocating nurses from across the world to UK and US hospitals.

As at this stage the site is just to provide information, the site only needed HTML and CSS, but was still a good way for me to learn more about making sites for clients.

67 Degrees

This is what I would class as my first website - a coffee shop site created for one of my coursework projects. I got to choose the topic and contents of the site, and wanted to test myself to my ability at an early stage of coding.

Before this, I had only made sites that would not fit mobiles properly, or that didn't feature anything too complex. However, after this site, I was able to start coding properly and began to understand what you can achieve with HTML, CSS and JavaScript.
